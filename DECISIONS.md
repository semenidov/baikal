## 2026-07-20 — ESLint + Prettier (flat config), формат применён, линт в CI

- **Контекст:** линтер и форматтер в проекте отсутствовали; по стандарту рабочего пространства (`_workspace/development.md`) обязательны ESLint (flat config) + Prettier.
- **Решение:** ESLint 10 (`eslint.config.mjs`): js recommended + typescript-eslint + eslint-plugin-astro + eslint-config-prettier; `no-explicit-any: error`. Prettier 3 (`.prettierrc.json`): printWidth 100, singleQuote, semi, plugin astro. Скрипты `lint`, `lint:fix`, `format`, `format:check`. Кодовая база отформатирована целиком. В CI добавлены шаги `npm run lint` и `npm run format:check` (до `check`/`build`).
- **Причина / trade-offs:** `no-undef` отключён для TS/Astro — глобальные TS-типы (`ImageMetadata`) проверяет TypeScript, иначе ложные срабатывания. Единоразовый `npm run format` даёт большой diff — выносить отдельным коммитом `chore: format`, не смешивая с контентом.
- **Статус:** принято

## 2026-07-20 — Заявка только через Telegram, без онлайн-оплаты и бронирования

- **Контекст:** нужно целевое действие для лендинга небольшого турагентства.
- **Решение:** единственный CTA — переход в Telegram; онлайн-оплаты/бронирования нет.
- **Причина / trade-offs:** проще и дешевле для маленького агентства, живой контакт вместо интеграции платежей; минус — нет автоматизации заявок и мгновенной оплаты.
- **Статус:** принято _(реконструкция)_

## 2026-07-20 — Весь контент в одном файле `src/data/site.ts`

- **Контекст:** тексты и данные нужно править без правок разметки.
- **Решение:** все тексты/данные вынесены в `src/data/site.ts`; компоненты только читают их.
- **Причина / trade-offs:** единая точка правки контента, ниже риск сломать вёрстку; минус — файл растёт и совмещает разные секции.
- **Статус:** принято _(реконструкция)_

## 2026-07-20 — Astro (SSG) как фреймворк

- **Контекст:** дефолт рабочего пространства — Next.js/Vite, но нужен статический лендинг.
- **Решение:** Astro со статической генерацией, 0 JS по умолчанию.
- **Причина / trade-offs:** для контентного лендинга — максимум скорости и SEO при минимуме JS; отклонение от дефолтного стека оправдано типом проекта.
- **Статус:** принято _(реконструкция)_

## 2026-07-20 — npm вместо pnpm

- **Контекст:** дефолт рабочего пространства — pnpm; в проекте `package-lock.json` и CI на `npm ci`.
- **Решение:** пакетный менеджер проекта — npm.
- **Причина / trade-offs:** причина в истории проекта не зафиксирована; менять на pnpm сейчас нецелесообразно (лок-файл и CI уже на npm).
- **Статус:** принято _(реконструкция)_

## 2026-07-20 — Инлайн CSS при сборке

- **Контекст:** render-blocking запрос внешних стилей на лендинге.
- **Решение:** `inlineStylesheets: 'always'` в `astro.config.mjs` — CSS встраивается в HTML.
- **Причина / trade-offs:** быстрее первый рендер; минус — стили не кэшируются отдельно между страницами (для одностраничника несущественно).
- **Статус:** принято _(реконструкция)_

## 2026-07-20 — Хостинг Vercel, авто-деплой из `main`

- **Контекст:** нужен простой хостинг статики с CI.
- **Решение:** Vercel, авто-деплой при push в `main`; GitHub Actions прогоняет `astro check` + build.
- **Причина / trade-offs:** нулевая настройка для статики + аналитика Vercel; привязка к платформе (vendor lock-in) минимальна для статического сайта.
- **Статус:** принято _(реконструкция)_
